# 架构说明

## 项目概览

`Emberwell Roguelike` 是一个基于 Godot 4 的实时格子动作轻肉鸽。玩家在 30 x 18 的程序生成地牢中逐层深入，核心循环为：

`主菜单 -> 生成楼层 -> 探索/战斗/收集 -> 清空敌人 -> 出口开启 -> 选择祝福 -> 下一层 / Boss 胜利 / 失败`

当前版本已经包含 5 层完整流程、敌人 AI、道具成长、陷阱、余烬碎片、小地图、威胁摘要、楼层祝福、存档继续和 Boss 二阶段。

## 运行入口

- 主场景：`res://scenes/main.tscn`
- 根节点：`RoguelikeMain (Node2D)`
- 主脚本：`res://scripts/main.gd`
- 目标分辨率：`1280 x 720`

`main.tscn` 保持轻量，只挂载主脚本。地图、角色、道具、特效、HUD、菜单、说明、暂停、祝福选择和结果面板都由 `main.gd` 动态创建。

## AutoLoad

- `GameData`：运行时分数、生命、层级进度、提示状态。
- `GameState`：菜单、说明、游玩、暂停、完成等全局状态。
- `AudioManager`：加载和播放 Kenney Interface Sounds 短音效。
- `ParticleManager`：生成拾取、受伤、死亡和强化反馈粒子。
- `SaveManager`：保存继续游戏进度和最高分到 `user://savegame.json`。

## 主脚本职责

`scripts/main.gd` 目前承担完整主循环，主要区域如下：

- 常量配置：贴图路径、敌人类型、楼层定义、祝福池。
- 运行状态：玩家属性、当前楼层、敌人/道具/宝箱/喷泉/陷阱数组。
- 场景创建：背景层、地图层、装饰层、物品层、实体层、特效层、UI 层。
- 地牢生成：随机房间、走廊连接、失败保底地图。
- 楼层填充：敌人、药水、金币、余烬、装备、宝箱、喷泉、陷阱。
- 玩家输入：连续移动、冲刺、攻击、等待、暂停、重开。
- 敌人 AI：追踪、游荡、近战、烛影侍从直线施法、Boss 半血召唤。
- 互动处理：拾取、开箱、饮泉、触发陷阱、出口完成。
- HUD：生命、攻击、冲刺、得分、目标、威胁摘要、小地图、日志。

## 地牢生成

每层从全墙体网格开始：

1. 随机尝试放置多个不重叠房间。
2. 按房间放置顺序连接中心点，生成走廊。
3. 第一间房作为玩家出生点，最后一间房作为出口。
4. 根据 `LEVEL_DEFS` 放置敌人、补给、装备、互动物和陷阱。
5. 随机补充不阻挡移动的装饰物。
6. 如果随机房间不足，使用 `_build_fallback_dungeon()` 生成保底地图。

## 实时格子规则

- 玩家逻辑坐标使用 `Vector2i`，渲染位置由 `_cell_to_world()` 转换。
- 按住方向键 / WASD / HJKL 会按固定节奏连续移动。
- Shift 会向当前方向冲刺最多 2 格，不能穿墙或穿过敌人。
- 冲刺经过陷阱会触发陷阱，陷阱造成 1 点伤害后失效。
- 撞向敌人会进行近战攻击，并进入短攻击冷却。
- Space 或 `.` 会停下脚步，方便重新观察局面。
- 敌人独立计时行动，因此玩家不移动时地牢仍在推进。

## 敌人与交互

- 黏液：慢速基础敌人。
- 蝙蝠：高速追击敌人。
- 烛影侍从：会沿同一行或列直线施法，墙体和其他敌人会挡住视线。
- 游魂：高感知范围追击敌人。
- 赤甲守卫：高血量高伤害近战敌人。
- 深井看守：第五层 Boss，半血后加速并召唤低血游魂。

互动物包括：

- 药水：恢复生命。
- 金币：增加分数。
- 余烬碎片：增加分数，每 3 个触发一次小共鸣。
- 短剑：提高攻击力。
- 盾牌：提高最大生命并恢复生命。
- 宝箱：随机给金币、治疗或攻击强化，开启后更换贴图。
- 喷泉：一次性治疗，使用后变暗。
- 尖刺陷阱：一次性伤害，触发后变暗并失效。

## UI 结构

右侧 HUD 以战术信息为优先级：

- 楼层名称和深度。
- 生命、攻击、冲刺冷却、得分、敌人数量。
- 威胁摘要：陷阱、宝箱、喷泉和余烬计数。
- 当前目标和出口状态。
- 小地图：显示地形、玩家、出口、敌人、道具、喷泉、宝箱和陷阱。
- 最近事件日志。

菜单层包括主菜单、探索指南、暂停面板、楼层祝福面板、失败/胜利面板。
