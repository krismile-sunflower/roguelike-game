# 架构说明

## 概览

当前项目主线是一个 Godot 4 实时格子 roguelike。主循环为：

`主菜单 -> 生成地牢层 -> 玩家连续移动/冲刺/攻击 -> 怪物持续行动 -> 清理敌人 -> 出口开启 -> 楼层祝福 -> 下一层 / 胜利`

## 运行入口

- 主场景：`res://scenes/main.tscn`
- 根节点：`RoguelikeMain (Node2D)`
- 主脚本：`res://scripts/main.gd`

`main.tscn` 保持很薄，只挂载主脚本。地图层、实体层、特效层、HUD、菜单、说明、暂停和结果面板都由 `main.gd` 动态创建。

## 核心脚本职责

- `scripts/main.gd`
  - 生成房间和走廊式地牢。
  - 放置玩家、出口、敌人、补给、装备、宝箱、喷泉和装饰。
  - 处理键盘输入、按住连续移动、Shift 冲刺、攻击、停下观察、重开和暂停。
  - 按独立计时器驱动敌人追踪、游荡和攻击。
  - 管理楼层祝福、Boss 二阶段、楼层完成、失败、胜利和 HUD 刷新。
- `scripts/game_data.gd`
  - 保存当前层、完成记录、分数、生命和提示计数等轻量运行数据。
- `scripts/game_state.gd`
  - 提供菜单、说明、游玩、暂停、完成等全局状态。
- `scripts/save_manager.gd`
  - 使用 `user://savegame.json` 保存继续游戏进度和最高分。
- `scripts/audio_manager.gd`
  - 加载并播放 Kenney Interface Sounds 中的短音效。
- `scripts/particle_manager.gd`
  - 生成拾取、受伤和敌人死亡的轻量代码粒子。

## 地牢生成

每层从全墙体网格开始：

1. 随机尝试放置多个不重叠房间。
2. 按放置顺序连接房间中心点。
3. 第一间房作为玩家出生点，最后一间房作为出口。
4. 根据楼层配置放置敌人、药水、金币、装备、宝箱和喷泉。
5. 随机补充不阻挡移动的装饰物。

## 实时格子规则

- 玩家按住方向键 / WASD / HJKL 会按固定节奏连续移动。
- 玩家按住方向时按 Shift 会短冲刺最多 2 格；冲刺不能穿墙、穿怪或越过阻挡。
- 玩家撞向敌人会攻击，并进入很短的攻击冷却。
- Space 或 `.` 会停下脚步，不会主动推进怪物；怪物本来就会持续行动。
- 敌人在感知范围内会按自己的移动冷却持续接近玩家。
- 敌人与玩家相邻时会按攻击冷却造成伤害。
- 清理所有敌人后出口开启；非最终层先弹出 3 选 1 楼层祝福。
- 宝箱和喷泉是独立互动物，不再作为纯装饰出现。
- 第五层 Boss 半血后会变色、加速并召唤低血游魂。
