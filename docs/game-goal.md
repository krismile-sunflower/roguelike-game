# 项目目标：Emberwell Roguelike

## 核心定位

把当前 Godot 项目推进为一个可玩的多关卡回合制 roguelike。玩家在程序生成的地牢中逐层深入，通过移动、近战攻击、拾取补给、利用装备成长和清理敌人完成每层目标。

## 当前可玩标准

- 有主菜单、继续游戏、说明、暂停、重开本层、失败、楼层完成和最终胜利流程。
- 至少 5 层可连续游玩的地牢，每层有不同敌人组合、房间数量、补给和风险。
- 地图由房间和走廊程序生成，玩家和出口分别位于首尾房间。
- 玩家行动后敌人行动，形成清晰的回合节奏。
- 敌人会在感知范围内追踪玩家，近身后造成伤害。
- 道具能改变局内状态：药水回血、金币加分、短剑加攻击、盾牌加最大生命。
- 清理本层敌人后出口开启，踩上出口进入下一层。
- 使用许可证清晰的素材，并在 `assets/licenses/SOURCES.md` 中记录来源。

## 当前素材方向

- 已导入 Kenney Tiny Dungeon 作为主要视觉素材，覆盖地面、墙体、角色、敌人、道具、出口和装饰。
- 已复用 Kenney Interface Sounds 作为交互、攻击、拾取、受伤、下楼和胜利反馈。
- 后续可以补充更完整的 UI、音乐、角色动画帧和更高辨识度的 boss 表现。

## 后续优先级

1. 用 Godot 编辑器完整跑通 5 层流程，修复任何运行时报错。
2. 拆分 `scripts/main.gd` 中的地图生成、实体、战斗和 UI 逻辑。
3. 增加小地图、可见范围或战争迷雾。
4. 加入更多敌人行为：远程、召唤、逃跑、巡逻。
5. 增加永久进度或局外解锁。
6. 设计正式标题、图标、导出配置和桌面构建。
