# 后续计划

## 立即验证

1. 用 Godot 编辑器运行 `res://scenes/main.tscn`。
2. 从主菜单开始新游戏，至少打到第三层。
3. 检查第一层是否仍然简单易懂，没有过早出现陷阱压力。
4. 在第二层测试尖刺陷阱、余烬碎片和烛影侍从直线施法。
5. 检查小地图是否能正确显示玩家、出口、敌人、道具、宝箱、喷泉和陷阱。
6. 清完一层敌人，确认出口点亮和祝福选择正常。
7. 打开 Debugger，确认没有脚本报错和缺失资源。

## 下一轮玩法优先级

1. 视野或战争迷雾：让探索更有 roguelike 张力，小地图只显示已探索区域。
2. 房间模板：增加宝库、陷阱房、治疗泉房、Boss 前厅等可识别空间。
3. Boss 专属机制：让深井看守不只靠高血量和召唤压制玩家。
4. 更多敌人行为：巡逻、守门、逃跑、召唤、远程蓄力提示。
5. 祝福池扩展：增加暴击、护盾、冲刺冷却、陷阱免疫、余烬收益等方向。
6. 局外进度：谨慎加入永久解锁，避免破坏短跑局平衡。

## 技术整理

1. 将 `scripts/main.gd` 拆分为地图、实体、战斗、交互和 UI 模块。
2. 将 `LEVEL_DEFS`、`ENEMY_TYPES`、`REWARD_POOL` 移动到独立资源或 JSON。
3. 增加可提交的 Godot smoke test，验证主场景、新局、第 2/3 层生成和关键互动。
4. 增加地图可达性检查，确保玩家出生点到出口始终连通。
5. 清理不参与主流程的历史脚本和场景，或移动到 `archive/`。
6. 为导出版本建立 preset，并记录 Godot 版本。

## 素材与呈现

1. 给玩家、敌人、出口和陷阱补充简单动画帧或轻量 Tween 表现。
2. 增加楼层环境音和低成本背景音乐。
3. 制作正式标题图标、桌面图标和启动封面。
4. 统一 UI 图标语言，为生命、攻击、冲刺、陷阱、宝箱和喷泉补齐小图标。
5. 为 Boss 准备更高辨识度的专属素材或调色。

## 平衡检查清单

- 第一层死亡率应低，主要用于学习。
- 第二层开始允许玩家因冲刺路线错误吃亏。
- 第三层以后要让喷泉和宝箱成为真实取舍。
- 第四层不应只靠堆敌人数变难，需要空间和陷阱共同施压。
- 第五层 Boss 半血后要紧张，但不应被召唤物直接淹没。
- 每层至少有一种“我可以做得更聪明”的决策点。
